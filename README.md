# Audio Transcriber

Локальный сервис транскрибации аудио и голосовых сообщений с интерфейсом на Gradio.

## Что умеет

- Загружать один или несколько файлов
- Поддерживать форматы `.mp3`, `.wav`, `.m4a`, `.ogg`, `.opus`, `.aac`, `.mp4`, `.mpeg`, `.webm`
- Распознавать речь через `faster-whisper`
- Сохранять текстовые результаты в `.txt`
- Добавлять таймкоды
- Останавливать обработку с сохранением текущего прогресса
- Работать как локальное приложение через готовые `dmg` и `exe` сборки
- Один раз спросить папку сохранения результатов и запомнить её

## Требования

- macOS, Windows или Linux
- Python 3.11+
- `ffmpeg` в `PATH`

## Установка на macOS

Установите `ffmpeg`:

```bash
brew install ffmpeg
```

Создайте и активируйте виртуальное окружение:

```bash
python3 -m venv .venv
source .venv/bin/activate
```

Установите зависимости:

```bash
pip install -r requirements.txt
```

## Установка на Windows

1. Установите Python 3.11+.
2. Установите `ffmpeg` и добавьте путь к `ffmpeg.exe` в `PATH`.
Пример типового пути:

```text
C:\ffmpeg\bin\ffmpeg.exe
```

3. Создайте и активируйте виртуальное окружение:

```bat
python -m venv .venv
.venv\Scripts\activate
```

4. Установите зависимости:

```bat
pip install -r requirements.txt
```

## Запуск

Кроссплатформенная версия:

```bash
python -B app.py
```

Оригинальный локальный файл:

```bash
python -B Транскрибатор.py
```

## Готовые сборки

Репозиторий поддерживает автоматическую сборку:

- `Transkribator-macos.dmg`
- `Transkribator-windows.zip` c `Transkribator.exe`

Сборки публикуются в GitHub Releases. Они создаются GitHub Actions и содержат встроенный `ffmpeg`, чтобы после скачивания не нужно было устанавливать его отдельно.

Важно:

- macOS-сборка не подписана Apple Developer ID, поэтому Gatekeeper может показать предупреждение при первом запуске.
- Windows-сборка тоже не подписана code-signing сертификатом, поэтому SmartScreen может показать предупреждение при первом запуске.
- Для проверки можно сравнить SHA256-файлы из релиза с загруженными архивами.

После запуска сервис будет доступен в браузере по локальному адресу, например:

```text
http://127.0.0.1:7860
```

## Как пользоваться

1. Откройте локальный адрес в браузере.
2. При первом запуске выберите папку, куда программа будет сохранять результаты.
3. Загрузите один или несколько файлов.
4. Выберите язык, режим результата и профиль скорости.
5. Нажмите `Транскрибировать`.
6. При необходимости нажмите `Остановить`, чтобы сохранить уже готовый прогресс.
7. Если захотите позже сменить папку, используйте кнопку `Изменить папку сохранения`.

## Структура проекта

- `app.py` — кроссплатформенная версия для GitHub
- `Транскрибатор.py` — исходная локальная версия
- `requirements.txt` — зависимости Python
- `.github/workflows/build-release.yml` — автоматическая сборка `dmg` и `exe`
- `transcription_outputs/` — результаты транскрибации
